# FANDEX v3 리빌드 설계서

## 1. v3 목표

FANDEX v3는 기존 mock market 사이트를 기반으로, K-pop 시장을 주식시장처럼 볼 수 있는 그래프 중심 데이터 플랫폼으로 확장한다.

v2는 아티스트별 가상 주가와 랭킹을 보여주는 포트폴리오형 mock 서비스였다.
v3는 다음 방향으로 발전한다.

* 모든 지수와 아티스트 주가를 꺾은선 그래프로 보여준다.
* 첫 화면에서 K-pop 종합지수의 흐름을 가장 먼저 보여준다.
* 실시간 K-pop 이슈를 인기순으로 정리한다.
* 아티스트 검색 기능을 제공한다.
* 아티스트 상세 페이지에서 소개, 주가 추이, 커스텀 지표, 최근 소식을 한눈에 보여준다.
* 콘텐츠랩을 AI 기반 SNS 콘텐츠 제작 도구로 발전시킨다.
* 전체 UI를 더 밝고 쉬운 한국어 중심으로 바꾼다.
* 실제 API와 DB를 연결할 수 있는 구조로 바꾼다.

## 4. v3 화면 구조

### 4-1. 홈 화면

홈 화면의 첫 번째 목적은 K-pop 시장 전체 흐름을 보여주는 것이다.

구성:

1. K-pop 종합지수 꺾은선 그래프
2. 오늘의 시장 요약
3. 실시간 K-pop 이슈 TOP 10
4. 아티스트 검색창
5. 상승/하락 기여 아티스트
6. 주요 신호 배지

홈 화면에서 가장 크게 보여줄 것은 K-pop 종합지수 그래프다.

### 4-2. 실시간 이슈 TOP 10

홈 화면 하단에는 K-pop 관련 이슈를 인기순으로 보여준다.

각 이슈는 다음 정보를 가진다.

* 순위
* 한 줄 헤드라인
* 관련 아티스트
* 관련 키워드
* 이슈 점수
* 기사 수
* 검색 증가율
* 최근 업데이트 시간
* 클릭 시 상세 요약 팝업

상세 팝업에는 다음을 보여준다.

* 이슈 요약
* 왜 중요한지
* 관련 아티스트
* 관련 뉴스 3~5개
* 지수 또는 아티스트 가격에 미친 영향
* 출처 링크

### 4-3. 아티스트 검색

검색창은 홈과 아티스트 페이지 모두에 배치한다.

검색 대상:

* 아티스트 한글명
* 아티스트 영문명
* 티커
* 멤버명
* 소속사
* 관련 키워드

초기에는 브라우저 안에서 검색한다.
나중에 데이터가 많아지면 검색 전용 API로 분리한다.

### 4-4. 아티스트 상세 페이지

아티스트 상세 페이지는 한 화면에서 핵심 정보가 보이게 만든다.

상단 구성:

* 아티스트 소개
* 현재 FANDEX 가격
* 등락률
* K-pop 종합지수 대비 성과
* 아티스트 주가 꺾은선 그래프
* 커스텀 인덱스 빌더

하단 구성:

* 최근 뉴스/소식 6개
* 각 소식 클릭 시 팝업
* 관련 키워드
* 공식 채널 링크
* 데이터 업데이트 시간

### 4-5. 커스텀 인덱스 빌더

커스텀 인덱스 빌더는 사용자가 직접 주가 산식을 바꿔보는 기능이다.

사용자는 아래 요소를 켜고 끌 수 있다.

* 음원 성적
* 음반 성적
* 유튜브 반응
* 공식 SNS 반응
* 검색량
* 뉴스량
* 해외 반응
* 팬덤 반응
* 소속사 규모

v3에서는 단순 체크박스보다 슬라이더 방식이 좋다.

예시:

* 검색량 30%
* 영상 반응 25%
* 뉴스량 15%
* SNS 반응 15%
* 해외 반응 15%

슬라이더를 움직이면 그래프가 즉시 다시 계산된다.

## 5. 시그널 페이지 처리 방향

현재 시그널 페이지가 단순한 테스트 페이지라면 독립 메뉴로 유지할 이유가 약하다.

v3에서는 두 가지 선택지가 있다.

### 선택지 A. 시그널 페이지 삭제

시그널을 독립 페이지로 두지 않고 홈과 아티스트 상세에 신호 배지로 흡수한다.

예시:

* 검색 급등
* 뉴스 급증
* 해외 반응 상승
* 팬덤 반응 증가
* 과열 주의
* 하락 주의

### 선택지 B. 시그널 페이지 재설계

시그널 페이지를 유지하려면 단순 숫자 카드가 아니라 경보 시스템처럼 보여야 한다.

각 시그널 카드에는 다음이 있어야 한다.

* 신호 이름
* 관련 아티스트
* 발생 시각
* 신뢰도
* 근거 데이터 3개
* 그래프에 미친 영향
* 관련 뉴스
* 해제 조건

v3 초기에는 선택지 A를 우선 적용한다.
데이터가 충분해진 뒤 선택지 B로 확장한다.

## 6. 콘텐츠랩 v3 방향

콘텐츠랩은 단순 문구 추천이 아니라 AI 콘텐츠 제작 도구가 되어야 한다.

목표 흐름:

1. 데이터 수집
2. 오늘의 이슈 선정
3. AI 기획안 생성
4. 카드뉴스 문안 생성
5. 숏폼 대본 생성
6. 썸네일 문구 생성
7. 사람이 검수
8. 발행 예약 또는 게시

초기에는 완전 자동 발행이 아니라, 사람이 승인한 뒤 게시하는 구조로 만든다.

콘텐츠랩 버튼 구조:

* 오늘의 아이템 발굴
* 콘텐츠 초안 만들기
* 카드뉴스 문구 만들기
* 숏폼 대본 만들기
* 발행 전 검수
* 복사 또는 예약 발행

v3에서는 최소 100개 K-pop 그룹을 다룬다.

각 아티스트는 다음 데이터를 가진다.

* id
* 한글명
* 영문명
* 티커
* 소속사
* 데뷔일
* 활동 상태
* 그룹 유형
* 세대
* 멤버
* 검색 키워드
* 공식 유튜브
* 공식 인스타그램
* 공식 X
* 공식 틱톡
* 공식 홈페이지
* 팬덤명
* 대표 국가 반응
* 데이터 수집 우선순위

초기 100팀은 수동 seed 데이터로 넣고, 이후 실제 API로 보강한다.

## 8. 실제 API 연결 계획

v3에서 바로 연결 가능한 데이터:

| 데이터        | API 후보                | 용도           |
| ---------- | --------------------- | ------------ |
| 뉴스         | 네이버 뉴스 검색 API         | 국내 이슈 수집     |
| 검색량        | 네이버 데이터랩              | 검색 추이 수집     |
| 글로벌 뉴스     | GDELT                 | 해외 이슈 수집     |
| 유튜브        | YouTube Data API      | 공식 영상 반응 수집  |
| 아티스트 메타데이터 | MusicBrainz, Wikidata | 기본 정보 보강     |
| 소속사 재무     | OpenDART              | 매출, 영업이익, 공시 |
| 콘텐츠 생성     | OpenAI API            | 콘텐츠 초안 생성    |

나중에 제휴가 필요한 데이터:

* 멜론 상세 차트
* 한터 초동 데이터
* 써클차트 세부 데이터
* 팬 플랫폼 멤버십 수
* 인스타그램/틱톡 세부 반응 데이터

## 10. 실시간 업데이트 구조

초기 상용화 구조:

1. Vercel Cron이 일정 주기로 데이터 수집 API를 실행한다.
2. API가 네이버, GDELT, 유튜브 등에서 데이터를 가져온다.
3. 가져온 데이터를 DB에 저장한다.
4. 가격과 지수를 다시 계산한다.
5. 화면은 최신 데이터를 가져와 그래프와 이슈를 갱신한다.

초기 업데이트 주기는 10~15분 단위가 현실적이다.
이후 트래픽과 비용을 보고 1분 단위로 줄인다.

## 11. UI 방향

v3 UI는 지금보다 더 밝고 쉬워야 한다.

원칙:

* 기본은 데이모드
* 나이트모드는 선택 기능
* 영어 비중은 10% 이하
* 메뉴명은 한국어 우선
* 숫자와 그래프는 크게
* 설명은 짧게
* 어려운 용어는 풀어서 쓰기
* 전문 용어에는 ? 도움말 달기

메뉴명 변경:

| 기존          | 변경       |
| ----------- | -------- |
| Market      | 시장       |
| Artists     | 아티스트     |
| Ranking     | 순위       |
| Signals     | 신호 또는 삭제 |
| Content Lab | 콘텐츠랩     |
| Methodology | 산정 방식    |
| About       | 소개       |

## 12. 산정 방식 페이지 개편

기존 Methodology 페이지는 어렵다.
v3에서는 메뉴명을 산정 방식으로 바꾼다.

설명 방식:

* 먼저 쉬운 말로 설명한다.
* 그다음 계산식을 보여준다.
* 용어 옆에 도움말을 붙인다.
* 예시를 들어 설명한다.

예시 문장:

FANDEX 가격은 실제 주식 가격이 아닙니다.
팬들의 반응, 검색량, 영상 반응, 뉴스량 등을 모아서 “요즘 얼마나 주목받고 있는지”를 숫자로 바꾼 값입니다.

## 14. v3의 1차 완료 기준

아래가 되면 v3 1차 완료로 본다.

* 홈에서 K-pop 종합지수 꺾은선 그래프가 보인다.
* 실시간 이슈 TOP 10 구조가 보인다.
* 아티스트 검색이 가능하다.
* 아티스트 상세에서 소개, 그래프, 커스텀 빌더가 한 화면에 보인다.

* 콘텐츠랩이 AI 콘텐츠 제작 흐름을 보여준다.
* 산정 방식 페이지가 쉬운 설명으로 바뀐다.

## 15. v3의 핵심 한 줄

FANDEX v3는 K-pop 시장 데이터를 그래프, 이슈, 아티스트별 지표, AI 콘텐츠 제작으로 연결하는 실시간 팬덤 데이터 플랫폼이다.
